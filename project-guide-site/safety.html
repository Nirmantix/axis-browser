Browser automation has sharp edges.

Keep secrets out of artifacts, treat protected sites as permission-gated, and choose shared or isolated browsers deliberately.

Hard rules

Artifacts stay under `.tmp/`

Screenshots, traces, scrapes, reports, verified runs, cookies, and generated credentials must not be committed.

Secrets stay user-local

Do not write API keys into repo files, `.env`, `.tmp/`, generated scripts, MCP snippets, or screenshots.

Protected sites need permission

Use BrowserAct, Notte, CloakBrowser, or Firecrawl on protected targets only when explicitly authorized.

Shared versus isolated browser mode

Mode Use when Risks
Shared Chrome on `9222` Real login, SSO, cookies, manual continuity, or cross-tool state matters. Can see real account data. Do not close user tabs from Playwright; use `disconnect()`.
Isolated Playwright Clean reproducibility, CI, destructive tests, or parallel agents matter. May not include real login state. Use test accounts and explicit test data.
Cloud/service browser Replay, live viewer, managed identity, or production browser workflows matter. Paid/API-backed. Credential setup must stay user-local and provider-specific.

Evidence rules

STEP_PASS

Use only when the check actually passed. Optional evidence paths are validated when present.

STEP_FAIL

Use when expected behavior did not happen. Include a screenshot path when useful.

STEP_SKIP

Use when a prerequisite is missing. Skipped work is honest, but it is not a pass.

No-go list

Do not invent credentials

If login blocks the task, ask the operator to log in or provide a test account through an approved local channel.

Do not hide failed checks

Report `STEP_FAIL` or `STEP_SKIP`. Do not remove plan checks to force a green validator.

Do not use stealth tools casually

CloakBrowser and protected-site flows are narrow, authorized-use paths, not defaults.

Do not nest LLM loops

Webwright is not part of the runtime stack. Host agents drive Axis Browser and browser-skill directly.
