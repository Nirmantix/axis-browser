Pick tools by job, not by novelty.

The Axis Browser workflow is tool-routed. Axis Browser is the compact diagnostic path, Browser Harness handles interactive browser work, and Playwright is added project-local for deterministic automation.

Core and common tools

Axis Browser

Fast shell-level diagnostics for shared Chrome and CDP workflows.

GitHub
  • Use for snapshots, refs, console, network, Lighthouse, perf, and heap.
  • Does not own an LLM loop. Host agents compose commands.
Browser Harness

Primary high-capability interactive browser tool in browser-skill.

GitHub
  • Use for visual control, iframes, odd widgets, and flexible browser operation.
  • Install from a durable editable checkout such as `$HOME/Developer/browser-harness`.
Playwright library

Deterministic browser automation library and test backend.

Official docs
  • Use for repeatable scripts, traces, API mocking, CI, visual regression, and cross-browser checks.
  • Install project-local when the target project needs those capabilities.
Microsoft Playwright CLI

Agent-oriented Playwright CLI, separate from project-local Playwright.

Official docs
  • Use when its CLI command surface or installed skills are useful.
  • Optional. It overlaps with Axis Browser and agent-browser.

Optional services and runtimes

Firecrawl CLI/MCP

Clean public-page markdown, crawl/search, structured extraction, and browser tools.

Codex MCP docs CLI GitHub
  • Use when a browser is overkill for public content.
  • Keep API keys in user-local env or MCP config.
BrowserAct

Authorized protected-site workflows, screenshots, bulk extraction, and Skill Forge.

Skills GitHub Skill Forge
  • Use only on systems owned, administered, or explicitly authorized for testing.
  • A prebuilt site skill is not authorization.
Notte

Cloud browser sessions, live viewer, workflow-code export, Functions, vaults, and personas.

Website Docs
  • Use when production browser workflows or cloud replay matter.
  • Credentials stay in Notte or user-local config.
CloakBrowser

Playwright/Puppeteer-compatible Chromium runtime for authorized anti-detection tests.

Website GitHub
  • Use as a narrow runtime, not a default browser automation strategy.
  • It is not a CAPTCHA solver or hosted proxy service.
agent-browser

Optional fallback CLI with browser control, install flow, and skills.

Skills docs
  • Use when it is the quickest available native route.
  • Run `agent-browser doctor` when an install looks stale.
Webwright

Microsoft research harness above Playwright where an LLM writes and verifies scripts.

GitHub Project site
  • Do not add it as an Axis Browser dependency.
  • Borrow verified-run and reusable-script discipline through browser-skill.
Browserbase skills

Separate Browserbase cloud/session ecosystem with its own CLI, API key, and platform workflows.

Skills GitHub Website
  • Use only when the task intentionally targets Browserbase cloud sessions or platform features.
  • Do not route normal Axis Browser work through Browserbase skills by default.

Fast picker

Task Use Why
Debug a local UI fast Axis Browser or Browser Harness Lowest overhead and direct browser state.
Write CI-grade automation Project-local Playwright Deterministic waits, traces, network interception, cross-browser.
Extract public documentation Firecrawl Clean content without browser overhead.
Work on protected targets BrowserAct, Notte, or CloakBrowser Only with explicit authorization and credential hygiene.
Use Browserbase cloud sessions Browserbase skills Separate ecosystem; only when Browserbase infrastructure is explicitly required.
Need reusable proof browser-skill verified run or craft workflow Structured evidence, STEP markers, and rerunnable scripts.
