Commands and entry phrases.

Use these when operating Axis Browser directly or when asking an agent to route work through browser-skill.

Axis Browser diagnostics

Use these for compact state from a real browser.

Open and inspect

axis-browser open URL
axis-browser snapshot

Routes to

Axis Browser bridge and chrome-devtools-mcp.

Expected output

Page title, URL, accessibility snapshot, generation-tagged refs.

Runtime debugging

axis-browser console --type error
axis-browser network --type fetch

Routes to

DevTools console and network surfaces.

Expected output

Error list, request list, IDs for `console-get` and `network-get`.

Evidence capture

axis-browser screenshot .tmp/screenshots/page.png --full-page

Routes to

Current selected page in the Axis bridge.

Expected output

Screenshot artifact under `.tmp/screenshots/`.

browser-skill scripts

Use these when the browser workflow needs setup, traces, reports, or validation.

Project setup

bash "$SKILL_DIR/scripts/setup.sh"

Routes to

Project-safe setup script.

Expected output

`.tmp/` artifact dirs, `.gitignore` entry, prerequisite check.

Tool check

bash "$SKILL_DIR/scripts/check-prerequisites.sh" --print-install-commands

Routes to

Read-only install guidance.

Expected output

Missing tools, auth presence signals, no installs, no secret values.

Machine setup

bash "$SKILL_DIR/scripts/check-prerequisites.sh" --install
bash "$SKILL_DIR/scripts/check-prerequisites.sh" --install --dry-run

Routes to

Permission-gated machine-level install audit.

Expected output

Core and optional tool actions approved, skipped, or blocked. No credential writes.

Health audit

bash "$SKILL_DIR/scripts/check-prerequisites.sh" --update
bash "$SKILL_DIR/scripts/check-prerequisites.sh" --update --dry-run

Routes to

Report-first version audit for shared tools.

Expected output

Current/latest versions, risk notes, pre-update record under `.tmp/axis-browser-health/` when applied.

Verified run

node "$SKILL_DIR/scripts/init-verified-run.mjs" --task "..." --slug run
node "$SKILL_DIR/scripts/validate-verified-run.mjs" --run .tmp/verified-runs/run

Routes to

Evidence-backed browser task workflow.

Expected output

Validated STEP markers with exit code 0, 1, or 2.

Prompt shortcodes and agent entry phrases

Use shortcodes for repeatable entry points and plain phrases for task-specific routing.

;absetup

Machine setup and install audit.

Routes to

`prompts/absetup.md` and `check-prerequisites.sh --install`.

Expected output

Tool status, approved/skipped installs, and credential gaps with values hidden.

;abcheck

Target-project readiness.

Routes to

`prompts/abcheck.md`, `setup.sh`, and `check-prerequisites.sh`.

Expected output

`.tmp/` readiness, available browser tools, and project-local Playwright recommendation.

;abuse

Browser task launcher.

Routes to

`prompts/abuse.md` and `SKILL.md`.

Expected output

Selected skill reference, selected tool, and evidence paths.

;abhealth

Monthly or biweekly health audit.

Routes to

`prompts/abhealth.md` and `check-prerequisites.sh --update`.

Expected output

Version audit, changelog impact, stale-doc findings, and proposed follow-up edits.

Debug

"Use browser-skill to debug why signup fails."

Routes to

Feature debugging, runtime debugging, Axis Browser or Browser Harness.

Expected output

Observed browser state, console/network evidence, root cause candidate.

Verify

"Use browser-skill to create a verified run proving checkout works."

Routes to

`references/verified-run.md`.

Expected output

`.tmp/verified-runs/<slug>/` with plan, findings, screenshots, validation JSON.

Craft

"Use browser-skill to craft a reusable local workflow for this browser task."

Routes to

`references/reusable-workflow.md`.

Expected output

Rerunnable `script.mjs` plus evidence and validation.

Maintain

"Update Axis Browser from upstream safely."

"Follow the private maintenance playbook."

"Sync this fork with upstream."

Routes to

Maintenance page and private local playbook.

Expected output

Upstream-state report, preserved fork deltas, validation results, and PR status when a PR is opened.
