Browser work without guessing.

Use Axis Browser for compact shared-Chrome diagnostics, then route bigger browser tasks through the browser-skill workflow for screenshots, traces, verified runs, reusable scripts, and safe tool selection.

Current workstation route

Set `BROWSER_SKILL_DIR` to the local browser-skill path, run setup in the target repo, then verify installed tools.

Fresh machine route

Run the setup audit, install machine-level tools, then add project-local Playwright only where deterministic automation is needed.

Prompt shortcodes

`;absetup`, `;abcheck`, `;abuse`, and `;abhealth` are thin wrappers around browser-skill and its scripts.

Need to debug a live app?

Attach to shared Chrome, inspect snapshots, console, and network before changing code.

Need evidence?

Use verified runs with STEP markers, screenshots, and validator exit codes.

Need the right tool?

Operating model

Axis Browser is a CLI. It does not own an LLM loop. Human operators and coding agents compose its commands inside a broader browser workflow.

1. Choose mode

Shared Chrome for real login state. Isolated browser for reproducible or destructive checks.

2. Observe first

Run snapshot, console, and network. Do not edit application code from a guess.

3. Drive the flow

Click and fill by generation-tagged refs, or switch to Playwright for deterministic multi-page scripts.

4. Save proof

Store screenshots, traces, reports, and verified-run artifacts under `.tmp/`.

What to do next

Set up a project

  • local Export `BROWSER_SKILL_DIR=/path/to/browser-skill`, run setup, verify tools.
  • fresh Install machine tools first, then install or copy browser-skill and run per-project setup.
  • output A project with `.tmp/` artifacts ignored and required browser tools checked.

Debug a feature

  • debug Open the app, snapshot, inspect console/network, reproduce the failure.
  • output A concrete failing request, exception, DOM state, or confirmed pass.

Verify a browser task

  • verify Scaffold a verified run, define checks, drive the browser, validate findings.
  • output STEP_PASS, STEP_FAIL, or STEP_SKIP with evidence.

Craft a reusable workflow

  • craft Prove the flow once, then refactor it into a parameterized script.
  • output A rerunnable local `script.mjs` plus evidence.

Maintain the fork

  • sync Check upstream, preserve fork deltas, validate, and merge through the protected PR flow.
  • output Upstream-change summary, validation results, and PR status.
No secret storage

Keep `.codex/`, MCP config, cookies, API keys, traces, and screenshots out of git.

No fake authorization

Protected-site work requires systems owned, administered, or explicitly authorized for testing.

No Webwright replacement

Borrow verified-run and reusable-script ideas. Do not replace Playwright or Axis Browser with Webwright.

Prompt entry points

The shortcodes are convenience entry points. They do not replace `SKILL.md`, and they do not create another routing system.

;absetup

  • scope Machine setup and install audit.
  • runs `check-prerequisites.sh --install`.

;abcheck

  • scope Target-project readiness.
  • runs `setup.sh`, then `check-prerequisites.sh`.

;abuse

  • scope Browser task launcher.
  • runs Loads `SKILL.md`; all routing stays there.

;abhealth

  • scope Monthly or biweekly health audit.
  • runs `check-prerequisites.sh --update` plus read-only content review.
